# DevTools 开发常用小工具集 — 产品需求文档 (PRD)

## 1. 项目信息

- **Language**: 中文
- **Programming Language**: Go（后端逻辑） + Wails v2（桌面壳 + 前端 UI React + Tailwind CSS）
- **Project Name**: `dev_tools`
- **原始需求**: 开发一个开发常用小工具集桌面应用，集合 JSON/YAML/Base64/XML 格式化与转换、随机字符串生成、时间戳转换、Cron 表达式解析等日常高频操作，要求 Windows 平台解压即用（便携版），界面简洁、响应快速。
- **架构变更**: 采用 **Wails v2** 单进程架构替代原计划的 Tauri v2 + Go sidecar 方案，简化部署并提升性能

---

## 2. 产品定义

### Product Goals

1. **一站式效率工具** — 将开发者日常高频的格式化、编码、转换等操作集中在一个窗口内完成，减少工具碎片化带来的时间浪费。
2. **零门槛便携体验** — Windows 解压即用，无需安装、无需配置环境，随时随地可用。
3. **即时反馈交互** — 输入即输出，所有操作在毫秒级响应，让开发者形成"粘贴→结果→复制"的肌肉记忆流程。

### User Stories

1. As a **后端开发者**, I want to **一键格式化/压缩 JSON** so that 我能快速排查接口返回的 JSON 数据结构问题。
2. As a **运维工程师**, I want to **解析 Cron 表达式并查看下次执行时间** so that 我能验证定时任务配置是否正确。
3. As a **全栈开发者**, I want to **将 YAML 转为 JSON** so that 我能方便地在配置文件和程序数据结构之间转换。
4. As a **前端开发者**, I want to **将时间戳与日期互转且支持多时区** so that 我能快速确认不同时区下的时间对应关系。
5. As a **开发者**, I want to **生成可配置的随机字符串** so that 我能快速获得测试数据或临时密钥。
6. As a **API 开发者**, I want to **编码/解码 URL 参数** so that 我能调试 HTTP 请求参数。
7. As a **安全工程师**, I want to **生成和验证 Hash 摘要** so that 我能验证数据完整性或生成签名。
8. As a **认证开发者**, I want to **解析 JWT Token** so that 我能调试和验证认证信息。
9. As a **开发者**, I want to **生成和验证 UUID** so that 我能获得唯一标识符或验证格式正确性。
10. As a **开发者**, I want to **测试正则表达式** so that 我能快速验证匹配规则和替换逻辑。

## 4. UI 设计稿

### 4.1 整体布局

采用**左侧侧边栏 + 右侧主内容区**的经典双栏布局：

```
┌──────────────────────────────────────────────────────┐
│  DevTools                                    ─ □ ✕  │ ← 标题栏（Tauri 自定义标题栏）
├────────────┬─────────────────────────────────────────┤
│  🔍 搜索   │                                         │ ← 侧边栏顶部搜索框（P2）
│ ─────────  │                                         │
│  📋 JSON   │      【当前工具内容区】                   │
│  📄 YAML   │                                         │
│  🔒 Base64 │      上方：输入区                        │
│  🏷 XML    │      ┌─────────────────────────┐        │
│  🎲 Random │      │  文本输入框 / 文件拖拽区   │        │
│  🕐 Time   │      │  (支持粘贴、拖拽文件)      │        │
│  ⏰ Cron   │      └─────────────────────────┘        │
│ ─────────  │      操作按钮行：[格式化] [压缩] [验证]  │
│  ⚙ 设置   │                                         │
│ ─────────  │      下方：输出区                        │
│  更多...   │      ┌─────────────────────────┐        │
│            │      │  结果展示区                 │        │
│            │      │  (格式化结果/错误信息)       │        │
│            │      └─────────────────────────┘        │
│            │      [📋 一键复制]                       │
│            │                                         │
├────────────┴─────────────────────────────────────────┤
│  状态栏：工具名称 | 操作结果提示                        │
└──────────────────────────────────────────────────────┘
```

### 4.2 侧边栏设计

- **宽度**: 固定 200px，可折叠为图标模式（60px）
- **工具列表**: 图标 + 文字名称，当前选中项背景高亮 + 左侧色条指示
- **分组**: 核心工具（P0）默认展示，扩展工具（P1/P2）顺序排列
- **底部**: 设置入口（主题切换、快捷键配置）

### 4.3 各工具页面设计

#### JSON 格式化
- **输入区**: 大文本框，支持粘贴 JSON
- **按钮行**: [格式化] [压缩] [验证] 三个操作按钮
- **输出区**: 格式化/压缩后结果；验证失败时显示 `Error at line X: 具体错误信息`，并高亮对应行

#### YAML 格式化
- **输入区**: 大文本框
- **按钮行**: [格式化] [验证] [转 JSON]
- **输出区**: 格式化结果或转换后的 JSON

#### Base64 编解码
- **输入区**: 双模式切换 — 文本模式（文本框） / 文件模式（拖拽上传区 + 文件路径显示）
- **按钮行**: [编码 →] [解码 ←]
- **输出区**: 编解码结果文本；文件模式解码后提供 [下载文件] 按钮

#### XML 格式化
- **输入区**: 大文本框
- **按钮行**: [格式化] [压缩] [验证]
- **输出区**: 同 JSON 格式化逻辑

#### 随机字符串生成
- **配置区**: 长度滑块/输入框（默认 16）、字符集勾选（☑大写 ☑小写 ☑数字 ☐特殊字符）、生成数量（默认 1）
- **按钮**: [生成]
- **输出区**: 生成的字符串列表，每条旁有 [📋 复制] 按钮

#### 时间戳转换
- **输入区**: 双向切换 — 时间戳→日期 / 日期→时间戳
  - 时间戳输入框 + 单位切换（秒/毫秒）
  - 日期时间输入框 + 时区选择下拉
- **输出区**: 转换结果 + 多时区对照表（UTC / 本地 / 用户选定时区）

#### Cron 表达式
- **输入区**: Cron 表达式输入框 + 常见预设模板下拉（如 `0 0 * * *` 每天零点）
- **按钮行**: [解析] [验证]
- **输出区**: 表达式含义描述 + 下次 5 次执行时间列表

#### URL 编解码
- **输入区**: 文本框输入 URL/参数
- **按钮行**: [编码] [解码] [解析参数]
- **输出区**: 编解码结果 / 解析后的参数表格（键值对）

#### Hash 摘要
- **输入区**: 文本框输入待哈希内容
- **按钮行**: [生成] [全部生成]
- **输出区**: 单个/全部哈希结果（MD5/SHA1/SHA256/SHA512），每条旁有 [📋 复制] 按钮

#### JWT 解析
- **输入区**: 文本框输入 JWT Token
- **按钮行**: [解析]
- **输出区**: Header/Payload/Signature 三部分展示，JSON 格式高亮，支持错误定位

#### UUID 工具
- **配置区**: 生成数量（默认 1）
- **按钮行**: [生成] [验证]
- **输出区**: 生成的 UUID 列表（每条旁复制按钮）/ 验证结果提示

#### 正则测试
- **输入区**: 正则表达式输入框 + 测试文本框
- **按钮行**: [测试] [替换]
- **输出区**: 匹配结果高亮显示 / 替换后结果文本

### 4.4 交互流程

1. 用户从侧边栏点击工具名 → 右侧即时切换为该工具页面（无刷新）
2. 在输入区粘贴内容或拖拽文件 → 点击操作按钮
3. 输出区即时显示结果 → 点击一键复制到剪贴板
4. 验证类操作失败 → 输出区显示错误定位信息

## 5. Open Questions

1. ~~**架构选择**: Tauri v2 vs Wails v2~~ → **已解决**：采用 Wails v2 单进程架构，简化部署，避免 Go sidecar 进程管理复杂性。
2. ~~**文件大小限制**: Base64 文件编解码是否需要限制最大文件大小？建议值？~~ → **已解决**：设置 10MB 上限，超过返回错误提示。
3. ~~**Cron 表达式标准**: 是否仅支持标准 5 位 Cron（分 时 日 月 周），还是需要支持 6 位/7 位扩展格式（如秒级 Cron）？~~ → **已解决**：支持标准 5 位 + 6 位秒级扩展格式。
4. **扩展工具机制**: P2 预留的插件机制是否在首版就需要设计接口，还是后续版本再加？
5. **快捷键方案**: 全局快捷键是否需要支持自定义绑定？默认快捷键映射表？
6. **YAML 转 JSON 的嵌套深度**: 是否需要处理超深层级 YAML？是否设置展示折叠？
7. **多语言支持**: 首版是否需要支持英文/中文切换？当前 UI 为中文。
8. **历史记录持久化策略**: 使用本地 JSON 文件还是 SQLite？缓存条目上限？
